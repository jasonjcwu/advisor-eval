# SWE-bench Evaluation

You are solving a real bug from an open source project. The repo is already checked out at the correct commit.

## Workflow
1. Read the problem statement carefully
2. Explore the codebase to understand structure
3. Identify the root cause
4. Make minimal, targeted edits
5. Verify the fix works (run tests if available)

## Rules
- Make minimal changes — fix the bug, don't refactor
- Don't add new files unless absolutely necessary
- Prefer editing existing code over rewriting
- Always check if there are existing tests you can run
